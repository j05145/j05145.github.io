Before Entering the Lagoon

What is XXE (External Entity Injection)

XML external entity (XXE) vulnerabilities (also called XML external entity injections or XXE injections) happen if a web application or API accepts unsanitized XML data and its back-end XML parser is configured to allow external XML entity parsing. XXE vulnerabilities can let malicious hackers perform attacks such as server-side request forgery (SSRF), local file inclusion (LFI), directory traversal, remote code execution (RCE), network port scanning, and denial of service (DoS).

What is OOB XXE?

Out-of-band XML external entity (OOB XXE) vulnerabilities are a type of XXE vulnerability where the attacker does not receive an immediate response to the XXE payload.

OOB XXE is sometimes confused with blind XXE due to the lack of a direct response.
With blind XXE, the attacker does not receive any response at all. Instead, they 
reconstruct sensitive data step-by-step based on the behavior of 
the 
targeted app, such as the web server and XML parser errors it generates.

The process for exploiting out-of-band XXE vulnerabilities is similar to using parameter entities with in-band XXE.

While in-band XXE can be used to conduct denial of service (DoS) and server-side request forgery (SSRF) attacks against web apps and APIs, the primary goal of OOB XXE is sensitive data exfiltration.

Email Sender (Very Important to Understand in this Engagement)

An email sender is an automated email account configured to send system-generated messages on behalf of a company or application. It is not typically linked to a real person, but rather used by back-end services, platforms, or cloud applications to 
perform automated communications.

Common Use Cases for an Email Sender:

Password recovery:
Sends password reset links or OTPs (One-Time Passwords) during account recovery workflows.

System notifications:
Alerts about suspicious login attempts, configuration changes, or billing events.

Transaction confirmations:
Sends purchase confirmations, payment receipts, or booking details.

Internal processes:
Delivers scheduled reports, automated alerts, or task assignment emails.

Client communication:
Sends welcome emails, appointment reminders, or automated follow-ups (often handled via email platforms like SendGrid or Mailchimp).

The attack chain began with a vulnerable endpoint hosted on an Azure App Service, which enabled the exfiltration of a web.config file through XXE. This configuration file exposed sensitive credentials, including access to a production database and a sender email account used for delivering OTPs (One-Time Passwords) during password recovery processes. These credentials facilitated further exploration and escalation within the target environment.

During a cloud security assessment of an application deployed in Azure App Service, an authenticated POST endpoint was found to be processing XML input without proper validation. The endpoint accepted user-supplied XML and was vulnerable to XXE attacks.

The structure of the XML request was deduced through public documentation and trial-and-error, which enabled the crafting of a malicious payload referencing an external DTD hosted on a controlled server.

Exploiting the XXE Vulnerability (OOB Exfiltration)

A payload was crafted to perform Out-of-Band (OOB) exfiltration using an external entity pointing to a listener server. This allowed reading internal files from the App Service environment, confirming the vulnerability by extracting:

  • C:\Windows\win.ini (for verification)
  • web.config (primary objective)

Secrets Found in web.config

The extracted configuration file contained:

  • Database connection strings with valid credentials
  • SMTP credentials for a sender email account

This sender email address was discovered to be used in critical flowsâ€”specifically for sending OTPs during password resets. With control over this account, an attacker could:

  • Intercept password reset tokens
  • Replay OTPs if they were not properly invalidated
  • Access user reset histories via the “Sent Items” folder

Security Impact

  • Unauthorized access to the production database
  • Complete takeover of an email sender used for OTP delivery
  • High risk of account takeover
  • Exposure of plaintext credentials in a cloud-hosted environment
